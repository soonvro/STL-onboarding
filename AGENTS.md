# 프로젝트 작업 규칙

이 문서는 저장소 루트 기준의 작업 규칙이다.
더 하위 디렉터리에 별도의 `AGENTS.md`가 있으면 해당 범위에서는 하위 문서가 우선한다.

## Justfile 우선 원칙

- 프로젝트 관련 명령 실행은 기본적으로 `Justfile`을 통해 수행한다.
- 특히 아래 작업은 먼저 `Justfile` 레시피 존재 여부를 확인한다.
  - 테스트 실행
  - 의존성 설치, 동기화, 업데이트
  - 헬스 체크, 상태 점검
  - 배포, 릴리스, 운영성 명령
- 적절한 레시피가 이미 있으면 원시 명령을 직접 실행하지 말고 반드시 `just <recipe>`로 실행한다.
- 적절한 레시피가 없으면 일회성으로 우회하지 말고, 먼저 `Justfile`에 재사용 가능한 레시피를 추가한 뒤 그 레시피를 사용한다.
- 새 레시피 이름은 역할이 바로 드러나게 짓는다. 예: `test`, `deps-install`, `deps-update`, `health-check`, `deploy`
- 기존 레시피 체계가 생기면 새 레시피도 그 체계를 따른다.
- 단순 탐색이나 파일 읽기처럼 프로젝트 운영 명령이 아닌 작업에는 이 규칙을 과도하게 적용하지 않는다.

## ADR 작성 요청 처리

- 사용자가 ADR 작성을 요청하면 작업 전에 `docs/adr/AGENTS.md`를 먼저 확인한다.
- ADR의 파일명, 템플릿 사용 방식, 본문 언어, 상태 표기, 인덱스 갱신 규칙은 `docs/adr/AGENTS.md`를 단일 기준으로 삼는다.
- ADR을 생성하거나 수정할 때는 `docs/adr/000_index.md`까지 함께 관리한다.

## 아키텍처 논의 처리

- 사용자가 아키텍처, 설계, 기술 선택, 의사결정, 트레이드오프 관련 요청을 하면 먼저 `docs/adr/AGENTS.md`를 확인한다.
- 아키텍처 논의는 단순 답변으로 끝내지 말고, ADR로 남길 가치가 있는 결정인지 함께 판단한다.
- 논의 결과가 장기적으로 참조할 결정으로 이어지면 `docs/adr/AGENTS.md`의 규칙에 따라 ADR 생성 또는 갱신을 진행한다.
- 사용자가 문서화까지 명시적으로 요청하지 않았더라도, 결정 기록이 필요하다고 판단되면 ADR 작성 또는 갱신을 제안한다.

## 우선순위

- 저장소 전반 작업에는 이 문서를 적용한다.
- `docs/adr/` 아래 작업에는 `docs/adr/AGENTS.md`를 반드시 함께 확인하고, 세부 규칙은 그 문서를 우선한다.
