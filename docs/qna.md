안녕하세요.
과제의 범위를 맞추기 위해 몇 가지 확인드리고 싶습니다.
정해진 답이 없다면 아래 가정으로 진행해보겠습니다.
편하실 때 답변 주시면 감사하겠습니다.

* 사용자가 등록 후 자신의 문의 상태를 조회하는 기능은 범위에 포함되지 않는 것으로 이해해도 될까요?
  -> 별도의 정답이 없다면: 사용자 측에는 등록 결과만 안내하고, 상태 조회와 관리는 관리자 페이지에서만 가능하도록 구현하겠습니다.

* 문의 등록은 로그인 없이 누구나 가능한 구조로 이해해도 될까요?
  -> 별도의 정답이 없다면: 공개 문의 폼으로 구현하겠습니다.

* 관리자 페이지는 인증 없이 구현해도 되는지, 아니면 최소한의 로그인/보호 장치가 필요한지 궁금합니다.
  -> 별도의 정답이 없다면: 복잡한 인증 시스템 대신 간단한 비밀번호 보호 수준의 최소한의 접근 제어를 두는 방향으로 진행하겠습니다.

* “처리중” 상태로 변경할 때는 별도의 외부 워크플로우 호출 없이 내부 상태만 변경되고, “완료됨”으로 변경될 때만 n8n 완료 처리 워크플로우가 호출되는 것으로 이해하면 될까요?
  -> 별도의 정답이 없다면: "처리중" 상태 변경도 Notion DB에 반영되도록 구현하겠습니다.

* 선택 과제의 유효성 검사는 어느 수준까지 기대하시는지 궁금합니다.
  -> 별도의 정답이 없다면: 일반적인 형식 검사 수준으로 이해하여, 필수값 여부, 공백-only 입력 방지, 형식 검사 정도를 적용하겠습니다.
