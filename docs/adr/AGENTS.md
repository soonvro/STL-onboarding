# ADR 작성 규칙

이 디렉터리에서 ADR(Architecture Decision Record)을 생성하거나 수정할 때는 아래 규칙을 따른다.

## 적용 범위

- 이 규칙은 `docs/adr/` 아래의 ADR 문서 전체에 적용한다.
- 주요 관리 대상 파일은 다음과 같다.
  - `_template.md`: ADR 본문 템플릿
  - `000_index.md`: 전체 ADR 인덱스
  - `NNN_*.md`: 실제 ADR 문서

## 아키텍처 관련 요청 처리

- 사용자가 아키텍처, 설계, 기술 선택, 트레이드오프, 구조 변경 관련 요청을 하면 먼저 `docs/adr/000_index.md`와 관련 ADR을 확인한다.
- 관련 ADR이 있으면 현재 요청이 기존 결정을 따르는지, 보완하는지, 변경하는지 먼저 판단한다.
- 관련 ADR이 없으면 현재 주제에 대한 명시적 결정 기록이 아직 없다고 보고 논의를 진행한다.
- 기존 ADR과 충돌하거나 방향을 바꾸는 요청이면 그 사실을 답변이나 작업 계획에서 분명히 언급한다.
- 아키텍처 논의 결과가 장기적으로 유지할 결정으로 이어지면 새 ADR 작성 또는 기존 ADR 갱신이 필요할 수 있음을 함께 언급한다.

## 작성 원칙

- 새 ADR은 반드시 `docs/adr/_template.md`를 복사해서 시작한다.
- 템플릿 안의 `{{ ... }}` 플레이스홀더는 모두 실제 내용으로 치환한다.
- 최종 ADR 문서에는 `{{` 또는 `}}`가 남아 있으면 안 된다.
- ADR 본문은 전부 한국어로 작성한다.
- `Status` 항목도 한국어로 작성한다. 예: `채택`, `대체됨`, `ADR 003 대체`
- `1. Title`에는 사람이 읽는 정식 제목을 적는다.
- 특별한 요청이 없는 한 `_template.md` 자체는 수정하지 않는다.

## 파일명 규칙

- ADR 파일명 형식은 `NNN_한국어_제목_요약.md`로 고정한다.
- `NNN`은 항상 세 자리 숫자다. 예: `001_로그인_세션_전략.md`
- 제목 요약은 한국어를 유지하고, 공백은 `_`로 바꾼다.
- 파일명에 불필요한 특수문자는 넣지 않는다.
- 새 ADR 번호는 기존 `NNN_*.md` 파일 중 가장 큰 번호의 다음 값을 사용한다.
- 번호 계산에서 `000_index.md`, `_template.md`, `AGENTS.md`는 제외한다.
- 실제 ADR이 하나도 없으면 `001`부터 시작한다.

## 인덱스 규칙

- ADR을 새로 만들거나 기존 ADR의 제목, 상태, 파일명이 바뀌면 `docs/adr/000_index.md`를 반드시 함께 갱신한다.
- `000_index.md`는 에이전트가 전체 아키텍처 결정을 빠르게 훑어보는 용도다.
- 인덱스는 아래와 같은 표 형식으로 유지한다.

```md
| 번호 | 제목 | 상태 | 링크 |
| --- | --- | --- | --- |
| 001 | 로그인 세션 전략 | 채택 | [NNN_한국어_제목_요약.md](./NNN_한국어_제목_요약.md) |
```

- 인덱스 행은 번호 오름차순으로 정렬한다.
- 인덱스의 `제목`은 ADR 본문의 `Title`과 일치해야 한다.
- 인덱스의 `상태`는 ADR 본문의 `Status`와 일치해야 한다.
- 어떤 ADR이 다른 ADR을 대체하면 새 ADR만 추가하지 말고, 필요하면 기존 ADR의 상태도 함께 갱신한다.

## ADR 작성 절차

1. 기존 ADR 파일을 확인해서 다음 세 자리 번호를 계산한다.
2. `_template.md`를 새 파일명 `NNN_한국어_제목_요약.md`로 복사한다.
3. 템플릿의 모든 `{{ ... }}`를 실제 한국어 내용으로 치환한다.
4. `Title`, `Status`, `Context`, `Decision`, `Consequences`를 모두 채운다.
5. `000_index.md`에 해당 ADR의 번호, 제목, 상태, 링크를 추가하거나 갱신한다.
6. 최종 문서에 플레이스홀더가 남지 않았는지 확인한다.

## 최종 점검 체크리스트

- 파일명이 `NNN_한국어_제목_요약.md` 형식을 따르는가
- 번호가 기존 ADR 다음 순번인가
- ADR 본문이 한국어로 작성되었는가
- `{{ ... }}` 플레이스홀더가 모두 제거되었는가
- `000_index.md`가 최신 상태로 갱신되었는가
- 인덱스의 제목, 상태, 링크가 ADR 본문 및 파일명과 일치하는가
